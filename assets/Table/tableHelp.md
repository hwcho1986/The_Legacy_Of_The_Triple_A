# PLAY.JSON 데이터 작성 가이드 (ver.0.8.1)

본 문서는 `PLAY.JSON`에서 사용하는 각 이벤트 타입별 기능과 파라미터(`Value01` ~ `Value05`)의 사용법을 안내합니다.

## 이벤트 타입 요약표

| 타입 (type) | 설명 | Value01 | Value02 | Value03 | Value04 | Value05 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| **bgcolor** | 배경색 배경 변경 | 색상 코드 (#000) | 페이드 시간(초) | - | - | - |
| **onbgimage** | 배경 이미지 출력 | 이미지 경로 | 효과 (fadein, blind) | 시간(초) | 레이어 번호 | - |
| **offbgimage** | 배경 이미지 제거 | 레이어 번호 | - | 시간(초) | - | - |
| **onbgm** | 배경음 출력 | 오디오 경로 | "loop" 여부 | 시작 지연(초) | - | - |
| **oncharacter** | 캐릭터 출력 | 캐릭터 ID | 감정/상태 | 위치 (center, left, right) | 효과 (fadein) | 시간(초) |
| **offcharacter** | 캐릭터 제거 | 캐릭터 ID | - | - | 효과 (fadeout) | 시간(초) |
| **speak** | 대사 및 강조 | 캐릭터 ID | 이름 (번역 키) | 대사 내용 | 보이스 경로 | 이름 색상 |
| **anieffect** | 애니메이션 효과 | 대상 (ID 또는 all) | 효과 이름 | 시간(초) | - | - |
| **oncolor** | 화면 오버레이 | 색상 코드 | 페이드 시간 | 투명도 (0.5) | - | - |
| **offcolor** | 오버레이 제거 | 페이드 시간 | - | - | - | - |
| **choice** | 선택지 분기 | 선택지 1 텍스트 | 이동할 Index 1 | 선택지 2 텍스트 | 이동할 Index 2 | 스탯 변경 등 |
| **quiz01** | 퀴즈 (주관식) | 퀴즈 질문 | 정답 리스트(배열) | 힌트 1 | 힌트 2 | 힌트 3 |
| **title** | 챕터 타이틀 | - | 노출 시간(초) | 텍스트 색상 | 타이틀 텍스트 | - |

---

## 상세 설명 및 예제

### 1. 텍스트 특수 태그
대사(`speak`)나 이름 등 모든 텍스트 필드에서 다음 태그를 사용할 수 있습니다.
-   **/n** 또는 **\\n**: 줄바꿈을 적용합니다.
-   **[RRGGBB]텍스트[-]**: 괄호 안의 헥스 코드로 텍스트 색상을 변경합니다.
    -   *예시: `[FF0000]경고[-]가 발생했습니다.`*

### 2. 주요 이벤트 예시

#### 캐릭터 등장 및 대사
```json
{
    "Index": 130,
    "type": "oncharacter",
    "Value01": "jung",
    "Value02": "normal",
    "Value03": "center",
    "Value04": "fadein",
    "Value05": 1.0
},
{
    "Index": 140,
    "type": "speak",
    "Value01": "jung",
    "Value02": "정성훈",
    "Value03": "드디어 도착했군.../n[AccentGold]유산[-]의 단서를 찾아보자.",
    "Value05": "#C5A059"
}
```

#### 애니메이션 효과 (Shake)
```json
{
    "Index": 160,
    "type": "anieffect",
    "Value01": "jung",
    "Value02": "shake",
    "Value03": 0.5
}
```

#### 주관식 퀴즈
```json
{
    "Index": 1020,
    "type": "quiz01",
    "Value01": "루마니아의 흡혈귀 전설 성의 이름은?",
    "Value02": ["브란성", "Bran Castle"],
    "Value03": "힌트 1 내용",
    "Value04": "힌트 2 내용"
}
```

---

> **참고**: 모든 파일 경로는 해당 HTML 파일을 기준으로 한 상대 경로여야 합니다. (예: `assets/images/...`)
